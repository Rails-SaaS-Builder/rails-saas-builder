# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Rails SaaS Builder, please report it responsibly.

**DO NOT** open a public GitHub issue for security vulnerabilities.

Instead, email **security@rails-saas-builder.com** with:

- A description of the vulnerability
- Steps to reproduce (if applicable)
- The affected gem(s) and version(s)

## What to Expect

- **Acknowledgement** within 48 hours of your report
- A fix timeline based on severity
- Credit in the release notes (unless you prefer to remain anonymous)

## Supported Versions

Only the latest release is actively supported with security fixes.

| Version | Supported |
|---------|-----------|
| Latest  | Yes       |
| < 1.0   | No        |
